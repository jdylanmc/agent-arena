## Summary

Build the **minimum viable foundation** for Agent Arena: a VS Code extension at `extension/` that integrates `@github/copilot-sdk@^0.1.8`, registers an Activity Bar view containing a webview "primary agent terminal", and proves the end-to-end round-trip from user prompt ‚Üí SDK ‚Üí model ‚Üí streamed response. Wire EI-1 telemetry through the SDK's built-in OpenTelemetry pathway with a normalization layer feeding the canonical Agent Arena event log. Wire EI-2 by capturing per-session manifests + content hashes that reference the SDK's session-state directory. Ship a binary yolo toggle (default OFF, per-workspace) that gates the SDK's mandatory `onPermissionRequest` handler through a typed `PermissionPolicy` abstraction designed for future per-tool policies. Provide a CI matrix on `ubuntu-latest` + `windows-latest` that uses an `SdkAdapter` test seam with a `FakeSdkAdapter` (the SDK ships no consumer-facing test harness ‚Äî research R-02). Land the wiki tree, two source-ingestion synthesis pages, and the log + harness schema docs as part of the same PR.

### Harness unload semantics (EI-2 ‚Äî closes deputy round-2 carry-forward)

EI-2 (`constitution.md:537-541`) requires that `loadHarness(A)` ‚Üí `unload()` ‚Üí `loadHarness(B)` work in the same process. CD-02 specified `save` and `load` but left `unload` ambiguous. This plan locks it down:

- **`unload()` is REPLACE semantics with the empty harness.** Internally implemented as `loadHarness(EMPTY_HARNESS)` where `EMPTY_HARNESS = { harness_version, agents: [], activeSessionId: null, sessions: [] }`. There is no separate `unload` code path; load is the only mutating verb.
- **In-memory state**: cleared in this order ‚Äî (1) abort current turn on every active `SdkSessionHandle`, (2) `disconnect()` every handle (releases in-memory; preserves on-disk), (3) clear the agent registry, (4) clear `activeSessionId`.
- **`workspaceState` (yolo state per CD-05)**: NOT cleared by load/unload. Yolo state is per-workspace user preference, not behavior-relevant runtime state per EI-2's definition. Loading harness B's `agents[].yoloMode` REPLACES the in-memory agent's yolo, but the persisted `workspaceState` entry is updated to match (so reload-after-restart preserves the loaded value).
- **SDK session directories on disk**: NOT touched by `unload()` or `load()`. The SDK is the system of record for session content (R-05). When `loadHarness(B)` references a session not on disk, load fails for that session entry only (`state: "unrecoverable"`, emit `aa.harness.session.unrecoverable.v1`) ‚Äî other sessions in B may still load successfully.
- **Round-trip verification**: a unit test `extension/test/unit/harness.unload.test.ts` exercises load A ‚Üí unload ‚Üí load B ‚Üí assert in-memory and on-disk states match B in the same process, with no process restart between transitions.
- **Cleanup of orphaned SDK session directories**: out of scope for the scaffold. Future spec may add `client.deleteSession(id)` cleanup at unload time when the user explicitly asks for it; do not bake destructive behavior into `unload`.

This semantics is binding for /speckit.implement and for `tasks.md`. The unit test is a **gate**: harness round-trip (FR-026 / SC-006) is not considered satisfied without the same-process unload test passing.
